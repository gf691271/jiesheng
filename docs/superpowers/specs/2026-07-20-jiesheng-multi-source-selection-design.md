# 接声多来源选择与音频信息增强设计

## 目标

在保持接声离线、单用途和易操作的前提下，改善大量录音的选择与排序体验：用户可以从音乐库、相册视频或系统文件夹添加素材；列表最多保留 20 个条目，并用足够完整的元数据帮助判断内容和先后顺序。

## 范围

### 选择入口

首页提供三个平级入口：

1. **音乐库**：申请必要的只读媒体权限后查询 Android MediaStore 音频索引。先显示包含音频的文件夹、相对路径和文件数量，进入文件夹后多选音频。
2. **相册**：调用系统视频选择器，只选择视频；接声提取视频中已有的音轨参与合并，不提供画面剪辑或重新编码视频的能力。
3. **文件夹**：继续使用 Android Storage Access Framework 系统文件选择器，覆盖 Download、云盘和未进入 MediaStore 索引的文件。

三个入口得到的素材统一进入同一个有序队列。应用不后台扫描、不建立媒体数据库，也不访问网络。

### 数量、去重和默认排序

- 队列最少需要 2 个条目才能合并，最多允许 20 个条目。
- URI 相同的素材视为重复，不重复加入。
- 每次添加素材后，整个队列按最后修改时间从早到晚排列，使连续录音尽可能直接形成可合并顺序。
- 修改时间相同的素材保持选择时的相对顺序。
- 修改时间未知的素材排在已知时间素材之后，并保持彼此的相对顺序。
- 用户仍可用上移和下移按钮手动微调；删除行为保持不变。
- 如果一次选择会超过 20 个，现有队列不改变，并明确提示数量上限，避免静默丢弃部分选择。

### 三行素材卡片

每个队列条目保留序号、上移、下移和删除按钮，信息区使用三行：

1. 完整文件名，自动换行，不使用省略号，不限制为单行。
2. 音频显示 `M4A · 12:36`；视频来源显示 `MP4（取音频）· 12:36`。格式优先取文件扩展名，缺少扩展名时回退到 MIME 类型，再缺失时显示“未知格式”。
3. 显示 `修改于 2026-07-20 14:36:08`；日期缺失时显示“修改时间未知”。日期按设备当前时区显示并精确到秒。

列表整体可滚动，长文件名增加单个卡片高度，但不挤压操作按钮。

## 架构与数据流

### 队列模型

`SelectedAudio` 在现有 URI、名称和时长之外增加：

- `formatLabel`：界面格式名称；
- `sourceType`：音频或视频取音频；
- `lastModifiedEpochMs`：可空的最后修改时间。

`AudioQueue` 继续作为不可变队列，统一处理去重、20 个上限、时间排序、移动和删除。时间显示使用独立的纯格式化函数，便于固定时区测试。

### 音乐库

新增轻量 `MediaLibraryRepository`，在用户进入音乐库时查询 MediaStore：

- Android 10 及以上使用 `RELATIVE_PATH` 分组；
- Android 8–9 在取得旧版只读存储权限后从媒体记录路径推导父目录；
- 查询显示名称、MIME、时长、修改时间、媒体 ID 和相对路径；
- 只保存当前页面所需的内存状态，不在本地缓存整库。

音乐库读取其他应用创建的音频时使用系统要求的权限：Android 13 及以上申请 `READ_MEDIA_AUDIO`，Android 8–12 申请限定到 API 32 的 `READ_EXTERNAL_STORAGE`。

### 相册与文件夹

相册入口使用系统视频选择器，不申请图片或视频全库读取权限。文件夹入口继续使用系统文档选择器并获取本次选择的 URI 访问权。

`DocumentMetadataReader` 将三个来源统一转换为 `SelectedAudio`。名称、MIME 和修改时间优先从 `ContentResolver` 查询，时长和是否存在音轨由媒体元数据或提取器验证。

### 合并

现有 Media3 `EditedMediaItemSequence.withAudioFrom(...)` 继续作为唯一合并实现。音频文件直接取音轨；视频只取其音轨。输出仍统一为 AAC 编码的 M4A，不引入 FFmpeg，不修改源文件。

## 状态与错误处理

- 用户拒绝音乐库权限时，显示简短说明并保留另外两个入口可用。
- 单个素材无法读取时，不把该素材加入队列，并显示其文件名和失败原因。
- 视频没有音轨时，提示“该视频没有可用音轨”，现有队列保持不变。
- MediaStore 条目已经被移动或删除时，在选择或合并阶段明确失败，不删除其他已选条目。
- 达到或超过 20 个时不部分接收本批选择，避免用户误判实际顺序。
- 合并过程的取消、目标文件清理和源文件不变规则沿用现有实现。

## 测试与验收

### 单元测试

- 第 20 个素材可加入，第 21 个被拒绝，原队列不变。
- 已知修改时间按从早到晚排序；相同时间保持稳定顺序；未知时间排在最后。
- 手动移动和删除仍保持其他素材元数据不变。
- 格式标签覆盖扩展名、MIME 回退和未知格式。
- 日期输出在固定时区精确到秒，未知日期显示固定文案。
- 合并按钮只在空闲状态且至少有 2 个素材时启用。

### Android 测试

- MediaStore 查询可按相对路径形成文件夹和数量。
- 文档 URI 能读取完整文件名、MIME、时长和修改时间。
- 包含音轨的视频与音频文件能够合并为单音轨 M4A。
- 没有音轨的视频被拒绝且不会破坏队列。
- 长文件名在卡片中完整换行显示，三个元数据行和操作按钮均可见。

### 发布验收

- API 35 模拟器通过所有单元测试、Android 测试和 release lint。
- 手工走通音乐库、相册和文件夹三个入口，各选择素材并确认时间默认排序。
- 正式签名 APK 可安装、冷启动和导出可播放的 M4A。
- GitHub Release 的 APK 与本地输出 SHA-256 一致。

## 非目标

- 不提供波形、裁剪、淡入淡出、混音或音量调整。
- 不读取视频画面、不输出视频、不选择照片。
- 不申请“所有文件访问”权限，不递归扫描全盘。
- 不后台扫描、不上传媒体、不增加账号或分析服务。
- 不保证云盘和厂商私有目录出现在音乐库；这些来源继续通过“文件夹”入口选择。
