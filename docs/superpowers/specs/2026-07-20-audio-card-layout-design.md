# 音频卡片布局微调设计

## 目标

让长文件名获得卡片的主要横向空间，消除右侧固定操作按钮造成的左侧拥挤与右侧留白。

## 范围

- 仅调整 `item_audio.xml` 的视觉布局。
- 顶部：序号圆点与完整文件名。文件名不省略，通常占一至三行；超过三行仍继续换行显示。
- 中部：格式与时长、修改时间两枚横向标签；窄屏允许标签自然换行，不截断时间。
- 底部：分隔线后放置等宽的上移、下移、删除小图标。沿用现有控件 ID、无障碍描述、启用状态与点击逻辑。

## 非范围

- 不改文件选择、20 项上限、时间排序、合并或 M4A 导出。
- 不增加新依赖、手势或详情页。

## 验收与测试

- 仪器测试先断言音频卡根布局为纵向，现有文件名完整显示与无省略规则不变。
- 测试验证操作按钮仍存在于底部操作行，保持原有 ID。
- 运行相关 Android 仪器测试；随后构建 debug APK 并在 API 35 模拟器核对长文件名、横向标签和底部三图标。
