# 接声 v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build, verify, sign, and publish a minimal Android app that orders 2–3 selected audio files and exports their sequential composition as M4A.

**Architecture:** One Android application module uses a `MainViewModel` for deterministic queue state and a narrow `AudioMergeEngine` boundary for Media3 Transformer. `MainActivity` only coordinates system document contracts and renders a single Android Views screen; output is first written to cache and then copied to the user-selected document URI.

**Tech Stack:** Kotlin 2.2.21, Android Gradle Plugin 8.9.2, Gradle 8.11.1, JDK 17, compile/target SDK 35, min SDK 26, AndroidX Views/ViewModel, Media3 Transformer 1.10.1, JUnit 4, AndroidX Test/Espresso.

## Global Constraints

- App name is `接声`; application ID is `com.frank.jiesheng`; version is `0.1.0` (`versionCode = 1`).
- Accept at most three user-selected `audio/*` documents and require at least two before export.
- Export AAC-LC audio in an MP4/M4A container; do not add FFmpeg.
- No network, broad storage, recording, account, analytics, or background-service permissions.
- Keep one app module and one media-engine interface; do not add a plugin system or editor timeline.
- All user-facing copy is Simplified Chinese.
- Release repository is public `gf691271/jiesheng` under Apache-2.0.

## File Map

- Gradle/root files: deterministic Android build and wrapper.
- `app/src/main/java/com/frank/jiesheng/AudioQueue.kt`: immutable queue rules only.
- `app/src/main/java/com/frank/jiesheng/MainViewModel.kt`: queue and merge UI state.
- `app/src/main/java/com/frank/jiesheng/DocumentMetadataReader.kt`: display name and duration lookup.
- `app/src/main/java/com/frank/jiesheng/AudioMergeEngine.kt`: merge contract and Media3 implementation.
- `app/src/main/java/com/frank/jiesheng/MainActivity.kt`: document contracts, export coordination, and rendering.
- `app/src/main/res/layout/activity_main.xml`: single-screen shell.

- `app/src/main/res/drawable` and `mipmap-anydpi-v26`: deterministic adaptive vector icon.
- `app/src/test`: pure queue, state, name, and composition tests.
- `app/src/androidTest`: URI metadata and end-to-end export checks.
- `README.md`, `LICENSE`, `THIRD_PARTY_NOTICES.md`: open-source and build documentation.
